Iris Paris Hub
Espace étudiant

Une application web qui centralise la vie étudiante IRIS Paris : informations, services, chat de promo, assistant IA, événements et profil personnel.

HTML CSS JavaScript Firebase optionnel LocalStorage
À dire On présente un site pensé comme un portail étudiant. L'idée est simple : un étudiant arrive sur une page publique, découvre les services, puis se connecte pour accéder à son espace personnel.
Utilité du site

Pourquoi ce site existe ?

CentraliserRéunir services, agenda, bons plans, contact et informations utiles au même endroit.
AiderDonner un assistant IA, une FAQ, une météo campus et des ressources concrètes.
ConnecterPermettre aux étudiants d'échanger par promotion grâce au chat privé.

Le site répond au problème des informations dispersées et rend la vie étudiante plus simple, plus lisible et plus interactive.

À dire On peut expliquer que l'objectif n'est pas seulement de faire une belle page. Le site doit être utile : il donne accès rapidement aux informations importantes et crée un espace réservé aux étudiants.

Architecture technique

Comment le code est organisé

index.html contient la structure HTML, la configuration CFG, les modales, le dashboard et le JavaScript principal.
style.css contient le design system : variables CSS, thèmes, grilles, animations, responsive et composants.
img/ contient les visuels du campus utilisés dans le hero et les sections.
Le projet est en HTML, CSS et JavaScript pur : pas besoin de framework ni de compilation pour le lancer.
Pour la personne code Elle peut expliquer que le projet est volontairement simple à déployer. On ouvre index.html dans le navigateur, et les fonctionnalités JavaScript s'exécutent côté client.
Authentification et données

Comptes, session et préférences

DBObjet JavaScript qui lit et écrit les utilisateurs, la session et les préférences dans localStorage.
openAuth()Ouvre la modale et choisit l'onglet connexion ou inscription.
emailRegister()Valide le formulaire, crée l'utilisateur et sauvegarde son profil localement.
openDash()Cache la vitrine publique, affiche le dashboard et initialise les widgets.
Limite à connaître : pour un vrai site en production, les mots de passe ne doivent pas être stockés dans localStorage. Ici, c'est un prototype pédagogique.
À dire On peut dire que localStorage permet de garder les données dans le navigateur. C'est pratique pour une démo et un projet scolaire, mais une vraie application utiliserait un backend sécurisé.
Chat de promotion

Temps réel avec fallback local

Verrou par promoL'utilisateur ne peut accéder qu'au chat lié à sa filière ou promotion.
FirebaseSi la configuration est active, les messages passent par Firebase Realtime Database.
FallbackSi Firebase ne répond pas, le site bascule en mode local avec localStorage.
Sécurité affichageLa fonction esc() échappe le HTML pour réduire le risque d'injection XSS dans les messages.
À dire Cette partie montre une vraie logique applicative : il y a une gestion d'accès, une présence en ligne, des messages, des réactions et un mode secours si le service externe n'est pas disponible.
IA, quiz et API

Des fonctions utiles au quotidien

Assistant IAAI_KB et AI_TOPICS donnent une réponse locale si l'API externe n'est pas configurée.
Quiz étudiantQUIZ_QUESTIONS contient les questions, points, réponses et explications.
Météo ParisinitWeather() interroge Open-Meteo, une API gratuite sans clé.
Compte à reboursinitCountdown() met à jour les jours, heures, minutes et secondes avec setInterval.
À dire Le but n'est pas d'ajouter des gadgets. Chaque module apporte quelque chose : aide aux révisions, culture tech, informations campus ou repère temporel avant un événement.
Interactions frontend

Les détails qui rendent le site vivant

IntersectionObserver déclenche les animations d'apparition quand les sections entrent dans l'écran.
setTheme() et setAccent() modifient les attributs data-theme et data-accent sur la balise html.
glowCard() calcule la position de la souris et transmet les coordonnées à CSS avec --mx et --my.
toast() affiche une notification temporaire pour confirmer une action utilisateur.
À dire Cette slide est parfaite pour montrer le lien entre design et code. Le design donne l'idée visuelle, et JavaScript rend l'interface réactive.
